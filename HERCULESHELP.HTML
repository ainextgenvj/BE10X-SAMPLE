Sales Quotation

Create and manage quotation documents that prospective buyers review before order confirmation.

Overview

  • Quotations / proforma invoices are issued to buyers for pricing approval.
  • Buyer acceptance converts the quotation into a Sales Order.
  • Sales Challan / Delivery Order and Invoice reference the original quotation.

Navigation

Select Account Receivable > Entry > Sales > Quotation.

Header details

  • Document Number: Enter quotation number. Use F1 to list existing numbers. Auto-filled if document numbering exists.
  • Document Date: Enter the quotation date.
  • Customer Name: Enter customer code or name. F1 lists customers and fills details automatically.
  • Party Quotation Number / Date: Enter buyer reference and its date.
  • Agent: Select agent using F1.
  • Currency Code: Choose required currency with F1; leave blank for local currency.

Product page

  • Product / Description: Select product using F1 and load its description.
  • Stock in Hand: Displays current stock availability.
  • Warehouse: Select godown or leave blank.
  • Quantity: Enter quantity. System prompts for selling unit and converted quantity when needed.
  • Rate: Picked from rate master based on customer policy; editable with rights.
  • Amount: Automatically calculated as Quantity × Rate. Press Enter to open term details and auto-calculate discounts.
  • Additional Description: Add extra product information.

Reference & address

  • Select taxes and percentages on the Terms page if required.
  • Enter Net Amount, Payment Type, Terms, Validity, and Remarks.
  • Address page auto-displays customer shipping/billing details, which may be edited.

Save: Click OK. Exit: Click Cancel.

Sales Order

Record confirmed customer orders and reserve stock before delivery.

Navigation

Select Account Receivable > Entry > Sales > Sales Order.

Header details

  • Document Number: Enter order acceptance number; use F1 lookup or auto-fill.
  • Document Date: Enter order date.
  • Customer Name: Enter customer using F1.
  • Party Order Number / Date: Optional buyer reference.
  • Agent: Select via F1.
  • Currency Code / Description: Choose currency and load its description.

Product page

  • Product: Select using F1.
  • Warehouse: Choose the dispatch godown or leave blank.
  • Quantity: Enter quantity; system prompts for alternate selling units if needed.
  • Rate: Pulled from rate master; editable with permission.
  • Amount: Calculated automatically, with term-based discount details available.
  • Additional Description: Enter any extra product information.

Reference

  • Enter Net Amount and Remarks. Remarks carry forward to the delivery order.
  • Enter terms, conditions, and address details as required.

Save: OK. Cancel: exit without saving.

Sales Despatch Note

Create delivery challans once the order is confirmed and goods are dispatched.

Navigation

Select Account Receivable > Entry > Sales > Sales Despatch Note.

Header details

  • Document Number: Challan number with F1 lookup and auto-fill.
  • Document Date: Challan date.
  • Customer: Select the customer for the dispatch note.
  • Quotation Details: Open quotation reference window.
  • Order Details: Open order reference window.
  • Manual DC Number / Date: Enter optional delivery note references.
  • Agent: Select via F1.
  • Currency Code: Select currency and load description.

Product page

  • Product, warehouse, quantity, rate, and amount details appear automatically.
  • Amount is calculated from Quantity × Rate.
  • Add additional description for the dispatch line as needed.

Reference page

  • Net amount and party order fields display automatically based on selection.

Save: OK. Cancel: exit without saving.

Sales Invoice

Raise the invoice after the goods have been delivered and dispatch note is complete.

Navigation

Select Account Receivable > Entry > Sales > Sales Invoice.

Header details

  • Document Number: Invoice number. Use F1 for lookup or auto-fill.
  • Document Date: Invoice date.
  • Customer Name: Select customer using F1.
  • Outstanding Dispatch Note: Select dispatch notes to invoice.
  • Manual Bill Number / Date: Enter optional bill references.
  • Agent: Select via F1.

Product page

  • Product, description, warehouse, quantity, rate and amount load from the selected dispatch note.
  • Rates can be changed if user rights allow it.
  • Use Add Description to insert an extended item note.

Reference page

  • Net amount is displayed automatically.
  • Enter reference challan number/date, party order number/date, transporter number, truck number, remarks, final destination, port/place of loading, country of origin, mode of transport, payment terms, validity of opening L/C, confirmation required, agent commission, insurance selection, specification, and customer bank details.
  • Billing and shipping addresses include district, state, country, pin code, and email data.

Save: OK. Cancel: exit without saving.

Return Credit Note

Issue a credit note after the customer returns goods and the return is recorded.

Overview

  • Customer account is credited once the Sales Return Credit Note is entered.

Navigation

Select Account Receivable > Entry > Sales > Return Credit Note.

Header details

  • Document Number: Credit Note number with F1 lookup.
  • Date: Credit Note date.
  • Customer: Automatically displayed when Sales Return reference is selected.
  • Agent: Displayed based on reference number.
  • Reference Bill: Enter the original sales bill for this return.

Product page

  • Details from the selected return challan are displayed automatically, including product lines, quantity, rate, amounts and policy details.
  • You may enter a lower returned quantity if needed.

Term & reference pages

  • Term details are picked from the Sales Return module if billing terms are defined.
  • Reference page displays relevant reference details.

Address page

  • Billing and shipping addresses are populated from master data and may be modified.

Save: OK. Cancel: exit without saving.

R/R In

Record returned goods and replacements coming back into stock.

Concept

  • Used when goods are returned on challan or invoice and need to be tracked.
  • Inventory-only returns are also supported without financial accounting.
  • Financial accounting is affected if charges are applied.

Navigation

Select Accounts Receivable > Entry > Sales > R/R In/Out.

Header details

  • Category: Choose In for inward goods or Out for outward goods.
  • Document No: Enter document number and use right-click for lookup.
  • Date: Enter document date.
  • Challan No / Date: Enter related challan number and date.
  • Bill No / Date: Enter related bill number or dummy number when required.
  • Customer: Enter customer or leave blank for cash sale. Blank entries prompt for further payment details.
  • Salesman: Default salesman displays from master data.
  • Class: Enter document class for branch/depot/type of order.

Product page

  • Product on Bill/Challan: Enter invoiced product. If using an invoice, only invoiced products are accepted.
  • Product In/Out: Enter returned product code for seconds or damaged goods.
  • Warehouse: Select receiving warehouse or leave blank.
  • Quantity: Enter returned quantity.

Reference page

  • Debit Note Number: Enter debit note number.
  • Remarks: Enter remarks carried forward from order or challan.

Address page

  • Billing and shipping addresses are auto-populated from master data and can be updated.

Save: OK. Cancel: exit without saving.

R/R Out

Record replacements or rejected goods sent back to customers.

Overview

  • Use when sending replacement goods or rejected items outward.
  • Supports inventory-only and accounting-affected outbound replacements.

Navigation

Select Accounts Receivable > Entry > Sales > R/R In/Out and choose Out.

Header details

  • Category: Out for outward delivery.
  • Document No: Enter document number and use right-click lookup.
  • Date: Enter document date.
  • Challan No / Date: Enter related challan and its date.
  • Bill No / Date: Enter related bill or dummy number.
  • Customer: Select customer or leave blank for cash sales.
  • Salesman: Default salesman displays and may be changed.
  • Class: Enter document class for order categorization.

Product page

  • Product on Bill/Challan: Only invoiced products are accepted when an existing invoice is selected.
  • Product In/Out: Enter returned product code for damaged or alternate items.
  • Warehouse: Select warehouse or leave blank.
  • Quantity: Enter quantity being returned or replaced.

Reference page

  • Debit Note Number: Enter debit note number.
  • Remarks: Auto carried forward from order or challan where applicable.

Address page

  • Modify billing and shipping addresses if required.

Save: OK. Cancel: exit without saving.

Retailer Invoice

Record retailer invoices for stockable products sold through cash or bank transit.

Navigation

Select Entry > Sales > Retailer Invoice.

Header details

  • Bill No: Enter sale bill number with right-click lookup.
  • Bill Date: Enter date.
  • Cash / Bank Code: Select using F1.
  • Customer Code: Enter or select customer. Leave blank for cash sale.
  • Cash / Bank: Enter the account for cash or bank credit.
  • Cheque No / Dated / On: Enter if using bank payment.
  • Party Name: Optional party name.
  • Due Date: Default calculated from credit days and modifiable.
  • Salesman: Defaults from master and can be changed.
  • Remarks: Enter relevant order remarks.
  • Batch No: Select from existing batch list.

Product page

  • Select product with lookup and enter quantity.
  • System prompts for selling unit conversion if needed.
  • Rate defaults from product master and can be updated.
  • Product amount is calculated automatically.

Terms and tax

  • Terms appear in code order and allow percentage entry with automatic amount calculation.
  • The tax popup shows GST/VAT/ST codes unless VAT was already answered as Y.

Save: OK. Cancel: exit without saving.
